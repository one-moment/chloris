# 결정 사항 — 보로 폐기·재고 인사이트 고도화

작성일: 2026-06-22
대상 화면: `/work/inventory/insights` (재고 인사이트)
관련 API: `app/api/work/inventory/metrics/route.js`

## 확정된 결정

| # | 항목 | 결정 | 비고 |
|---|---|---|---|
| D1 | 추이 단위 | **일·주·월 모두 지원**, **지점별** 비교 가능 | 주 = 월요일~일요일 |
| D2 | 폐기율 기준 | **가액 기준 유지** (폐기가액 ÷ 입고가액) | 수량 기준 폐기율 신설 안 함 |
| D3 | 구분(category) | **하드코딩 상수 고정 유지** | 마스터화/편집화면 만들지 않음 |
| D4 | 집계 대상 | **최종제출(`status:"submitted"`) 기록만** | draft·반려 건 제외 |
| D5 | 사유별 표기 | **비중(%) 표시로 변경** | 현행 건수·가액 원숫자 → 전체 대비 비중 |

## 이번 범위 (할 것)

1. 기간 버킷팅: 일/주/월 단위 집계 (월별·주별·일별 폐기율·입고가액·폐기가액)
2. 추이(시계열) 차트: 일/주/월 토글, 지점 "전체"면 지점별 시리즈 비교, 특정 지점이면 단일 시리즈
3. 품목별 집계: 품목별 폐기량(수량)·폐기가액, 폐기가액 높은 순 정렬, 표 형태
4. 구분(category)별 수치 화면 노출 (API가 이미 계산하는 `byCategory` 표시만 추가)
5. 사유별 표기를 비중(%)으로 변경

## 이번 범위 아님 (건드리지 않음)

- 구분(category) 마스터화/편집화면
- 마스터 행 "완전삭제" 기능 (현행 소프트 비활성 유지)
- 수량 기준 폐기율
- draft·반려 건 집계 포함
- 폐기/입고 입력 화면, 마스터 화면, 시트 내보내기 로직

## 기술 제약 / 그라운딩

- **DB 스키마 변경(마이그레이션) 불필요** — 전부 읽기 측 집계. 원천(DisposalBatch/Line, StockInDelivery/Line)에 날짜·품목·구분·지점·가액 존재.
- **새 차트 라이브러리 도입 금지** — `modules/analytics/ui/SalesDashboard.jsx`의 무의존성 인라인 SVG 차트 + day/week/month `ToggleGroup` 패턴 재사용.
- **버킷 키 규약** — `app/api/work/analytics/metrics/route.js`의 `bucketKey`(UTC 기준, 주=월요일 시작, 월=1일)와 동일 규약 사용해 일관성 유지.

## 진행 방식

- 코드 착수 전 구현 계획 보고 → "진행해" 승인 후에만 착수.
- 새 git 브랜치에서 작업, main/develop 직접 푸시 금지.
- 구현 후 직접 실행·확인, 테스트 통과 후 한국어 커밋 메시지로 커밋.
